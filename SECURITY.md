# Security Policy

## Supported Versions

Security fixes are applied on the current mainline only.

## Reporting a Vulnerability

Do not open a public GitHub issue for a suspected security problem.

Report it privately to the maintainer through GitHub's private security reporting flow if it is enabled for this repository. If that flow is unavailable, contact the maintainer directly through a private channel before public disclosure.

Include:

- affected files or workflow area
- reproduction steps
- impact assessment
- any suggested mitigation

This repository is maintained on a best-effort basis and does not offer a formal response SLA.
